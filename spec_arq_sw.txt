Atue como um Arquiteto de IA e Engenheiro de Software Sênior especializado em CrewAI, Streamlit e Engenharia Financeira (Value Investing).

Objetivo: Desenvolver uma aplicação completa em Python utilizando o framework CrewAI e Streamlit para automatizar o processo de triagem e análise qualitativa de ações da B3, mimetizando uma equipe de analistas de equity research.

1. Infraestrutura e Inicialização
Variáveis de Ambiente: O script deve ler os arquivos locais keys.txt e openrouter-key.txt para extrair e configurar as chaves OPENROUTER_API_KEY e SERPER_API_KEY, respectivamente.

LLM Setup: Inicialize o modelo utilizando a biblioteca do OpenRouter (via langchain_openai.ChatOpenAI ou a classe nativa do CrewAI).

Ferramentas (Tools): Configure o SerperDevTool para buscas na internet. Instrua os agentes a usarem operadores de busca avançados (como site:fundamentus.com.br e site:investidor10.com.br) para garantir que os dados venham das fontes corretas.

3. Arquitetura da Crew (Agentes e Tarefas)
Agente 1: O Triador Fundamentalista (Screener)

Role: Analista de Triagem Quantitativa B3

Goal: Filtrar o setor/subsetor escolhido e encontrar os 5 melhores ativos com base em eficiência e múltiplos de preço.
Dica: Para realizar os filtros utilizar as urls:
- https://fundamentus.com.br/resultado.php?setor="Código_do_setor"
- https://fundamentus.com.br/resultado.php?segmento="Código_do_segmento"
Onde os códigos de setor e segmento são os apresentados na estrutura de dados acima.

Task: Simular a triagem de dados no portal fundamentus.com.br para o setor/segmento enviado pelo usuário. Aplicar o seguinte algoritmo estrito:

Descartar imediatamente empresas com indicador P/L (Preço/Lucro) negativo ou zerado.

Ordenar as empresas restantes pelo indicador ROE (Return on Equity) de forma decrescente (do maior para o menor).

Cruzar e ponderar com o Dividend Yield (DY), priorizando os maiores valores positivos.

Retornar uma lista limpa com exatamente os Top 5 Tickers resultantes.

Agente 2: O Auditor Temporal (Análise Histórica)

Role: Auditor de Consistência Financeira

Goal: Validar se a rentabilidade das empresas do Agente 1 é recorrente ou um pico sazonal.

Task: Receber os top 5 tickers do Agente 1. Buscar no no portal fundamentus.com.br o histórico de gráficos temporais do ROE de cada uma deles. Analisar a consistência dos Indicadores Qualitativos nos últimos trimestres e anos. Filtrar e reordenar a lista, mantendo apenas as empresas que apresentem um ROE constante ou crescente ao longo do tempo.

Agente 3: O Investigador Governamental e de Governança

Role: Especialista em Governança Corporativa e Estrutura Acionária

Goal: Avaliar os riscos societários e o alinhamento de interesses dos controladores das empresas validadas pelo Agente 2.

Task: Investigar a composição acionária das empresas remanescentes na internet (simulando no portal fundamentus.com.br). Mapear e responder de forma ultra objetiva para cada ticker:

Qual é o percentual do Free Float ON (Ações Ordinárias em circulação)? (Validar se é alto).

Existe algum acionista majoritário com concentração excessiva de controle?

Existem sócios governamentais ou economia mista (risco político/estatal)?

Agente 4: O Comitê de Alocação (CIO)

Role: Diretor de Investimentos (Chief Investment Officer)

Goal: Consolidar os dados quantitativos, históricos e de governança em um relatório executivo final.

Task: Reunir as descobertas dos agentes 1, 2 e 3. Montar um Ranking final das empresas mais promissoras para investimento, justificando a posição de cada uma com base no equilíbrio entre ROE estável, Dividend Yield robusto, Free Float alto e baixo risco de interferência estatal.

4. Requisitos de Interface (Streamlit UI/UX)
Visual: Design minimalista e profissional, com tipografia limpa e paleta de cores voltada para o mercado financeiro (tons apropriados para usuários com dificuldades visuais).

Componentes de Entrada: Um st.selectbox obrigatório para o Setor. Ao ser selecionado, um segundo componente (st.selectbox ou st.multiselect) opcional deve aparecer renderizando os segmentos/subsetores mapeados no dicionário de dados.

Feedback de Execução (UX Fantástico): Utilize o componente st.status() do Streamlit para encapsular a execução da Crew, exibindo dinamicamente logs de progresso customizados para o usuário (ex: "Agente 1: Aplicando filtros fundamentalistas no Fundamentus...", "Agente 2: Analisando consistência histórica do ROE no portal ...").

Componentes de Saída: Exibir o ranking do Agente 4 usando tabelas interativas (st.dataframe) e detalhar a governança de cada empresa dentro de blocos expansíveis (st.expander) formatados em Markdown limpo.

Forneça o código Python completo, modular, com tratamento de erros para arquivos e chamadas de API, e com todos os comentários explicativos em português.
